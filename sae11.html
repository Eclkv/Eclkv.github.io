SAE 3.01 – Prise Connectée Intelligente Wifi

Conception et réalisation d'une prise électrique connectée en Wifi : schéma électronique sur EasyEDA, broker MQTT Mosquitto sur Raspberry Pi, serveur Django, interface web temps réel et capteur de température DS18B20.

ESP8266 MQTT Raspberry Pi Django Python Arduino EasyEDA

Contexte et Objectifs

Ce projet en groupe de 4 étudiants avait pour but de reproduire une prise électrique connectée de type CHACON — contrôlable via Wifi, depuis un bouton physique, une interface web et une application Android. L'ensemble du système repose sur un Raspberry Pi hébergeant un broker MQTT privé Mosquitto et un serveur web Django.

🎯 Objectif global

Concevoir l'électronique d'une prise connectée Wifi, implémenter la communication via le protocole MQTT, et proposer des interfaces de contrôle (web + Android) permettant d'allumer/éteindre des appareils à distance et d'afficher leur état en temps réel ainsi que la température ambiante mesurée.

Le microcontrôleur ESP8266 est le cœur de la prise : il gère le relais d'alimentation, la LED d'état, le bouton physique et le capteur DS18B20, tout en publiant et souscrivant aux topics MQTT. Le Raspberry Pi centralise les messages et sert l'interface web via Django.

Cahier des Charges — Étapes du Projet

✅ CdC 1 — Base

Schéma électronique + Serveur Web

Proposition du schéma structurel de la carte électronique sur EasyEDA, en justifiant les composants selon les critères de taille, poids, consommation et prix. Composant principal retenu : ESP8266 pour son Wifi intégré et son faible coût (~3€).

Implémentation du serveur web Django sur Raspberry Pi, capable de commander 2 prises en ON/OFF avec retour d'état en temps réel. Des LEDs physiques sur le RPi indiquent visuellement l'état de chaque prise.

⚡ CdC 2 — Sécurité & Configuration

Application Android + Broker Mosquitto + WiFiManager

Développement d'une application Android reprenant les fonctionnalités de la page web (contrôle ON/OFF, affichage des états en temps réel). Sécurisation de l'accès web par login et mot de passe.

Installation et configuration d'un broker MQTT Mosquitto privé sur le RPi. Utilisation de la bibliothèque WiFiManager (tzapu) pour permettre la configuration du réseau Wifi lors de la première installation (mode AP → STA).

🌡️ CdC 3 — Fonctionnalités Avancées

Contrôle global + Capteur DS18B20

Ajout d'un bouton "Tout allumer / Tout éteindre" sur l'interface web et Android, se synchronisant automatiquement si toutes les prises passent dans le même état via des actions individuelles.

Intégration du capteur de température DS18B20 (protocole One-Wire) avec affichage en temps réel de la température sur l'interface web et l'application Android.

Extraits de Code

Client MQTT Python souscrivant au topic d'état de la prise :

import paho.mqtt.client as mqtt BROKER = "lumenyana.local" PORT = 1883 TOPIC = "prise1/etat" def on_connect(client, userdata, flags, rc): if rc == 0: client.subscribe(TOPIC) # Souscription dès la connexion def on_message(client, userdata, msg): etat = msg.payload.decode() # "ON" ou "OFF" # → mise à jour BDD + interface web client = mqtt.Client(mqtt.CallbackAPIVersion.VERSION1) client.on_connect = on_connect client.on_message = on_message client.connect(BROKER, PORT, 60) client.loop_forever()

Rafraîchissement automatique de l'interface web toutes les 2 secondes :

async function refreshEtats() { const response = await fetch("/get_etats/"); const data = await response.json(); for (const [nom, etat] of Object.entries(data.etats)) { const btn = document.getElementById("btn-" + nom); if (etat === "ON") { btn.classList.replace("off", "on"); btn.innerHTML = "⚡ Éteindre"; } else { btn.classList.replace("on", "off"); btn.innerHTML = "💡 Allumer"; } } // Mise à jour des températures for (const [nom, temp] of Object.entries(data.temperatures)) { const el = document.getElementById("temp-" + nom); if (el && temp !== null) el.querySelector("span").textContent = temp + " °C"; } } setInterval(refreshEtats, 2000); // Auto-refresh toutes les 2s

Stack Technique

🔧 ESP8266 (NodeMCU) — Microcontrôleur Wifi intégré, programmé via IDE Arduino + PubSubClient MQTT
🌡️ DS18B20 — Capteur de température One-Wire, bibliothèque MAX31850 DallasTemp (Adafruit)
📡 Mosquitto — Broker MQTT open-source privé hébergé sur Raspberry Pi
🐍 paho-mqtt — Client MQTT Python pour la liaison broker ↔ Django
🌐 Django — Framework web Python : vues, templates, API JSON d'états
📱 Android Studio — Application mobile de contrôle ON/OFF et affichage températures
📐 EasyEDA — Logiciel de schéma électronique pour la conception de la carte PCB
⚡ Relais + Transistor — Commande du circuit 230V depuis l'ESP8266 (isolation galvanique)

Composants Électroniques Retenus

📶 ESP8266

Microcontrôleur principal : Wifi 802.11 b/g/n intégré, compact, faible consommation et très économique (~3€). Standard de l'IoT embarqué.

⚡ Relais 5V

Commandé par transistor pour basculer l'alimentation 230V. Isolation galvanique entre la partie commande basse tension et la partie puissance.

🌡️ DS18B20

Capteur numérique One-Wire, précision ±0,5°C, plage -55°C à +125°C. Un seul fil de données pour plusieurs capteurs en cascade.

🔋 Convertisseur AC/DC

Module PCB alimenté en 230V secteur, sortie 5V régulée pour l'ESP8266 et les périphériques. Sélectionné sur Farnell.

Ce projet m'a donné une vision concrète de l'IoT : comment un objet physique communique avec un serveur central, comment les interfaces se synchronisent en temps réel, et comment une architecture simple (ESP8266 + MQTT + Django) peut aboutir à un produit réellement fonctionnel, proche des objets connectés du marché.

Travailler à 4 sur des tâches parallèles a aussi renforcé mes compétences en coordination et en intégration de sous-systèmes développés indépendamment — une réalité du travail en équipe IT.

💡 Ce que j'ai retenu

Le protocole MQTT est élégamment simple et redoutablement efficace pour l'IoT : un broker central, des topics clairs, et n'importe quel client (Python, Arduino, Android) peut publier ou souscrire en quelques lignes. C'est cette légèreté qui en fait le standard incontournable des objets connectés embarqués.

Project Summary (English)

🌍 SAE 3.01 – Smart WiFi Connected Plug

Duration: 23 hours | Team: Group of 4 | Semester: 3rd

This project involved designing and building a WiFi-connected smart plug similar to commercial products like the CHACON smart plug. The system spans hardware design, embedded firmware, and a full web/mobile control interface.

Architecture: ESP8266 microcontroller (WiFi + relay + DS18B20 temperature sensor) communicating via MQTT with a private Mosquitto broker on a Raspberry Pi. A Django web server provides a real-time control interface; an Android app mirrors all functionality.

Key Features: ON/OFF control of 2+ plugs with real-time state feedback, physical push button + LED indicator, live temperature monitoring, login-secured web interface, and WiFiManager for first-boot network setup (AP mode → STA mode).

Outcome: A fully functional connected plug prototype — from PCB schematic (EasyEDA) to working software stack — deployed on a Raspberry Pi and controllable from web browser and Android.
